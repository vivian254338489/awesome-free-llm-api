# Inclusion and Exclusion Criteria

This document explains the criteria used for including or excluding providers from the awesome-free-llm-api list.

## Included Providers

Providers are included if they meet ALL of the following:

- **Legitimate official free tiers**: Providers must offer genuine free access through official channels. This includes:
  - Free tiers with defined limits (e.g., requests per day, tokens per month)
  - Trial credits for new accounts
  - Promotional offerings from established companies

- **Legitimate trial offers**: Time-limited trials from verified providers with:
  - Clear terms and conditions
  - No requirement to provide payment information (unless explicitly stated)
  - Official documentation

- **Open-source self-hostable alternatives**: Projects that allow running AI models on private infrastructure, including:
  - Open-weight models with usable licenses
  - Self-hosted inference solutions
  - Clear documentation for deployment

- **OpenAI-compatible developer tools**: Services that provide OpenAI-compatible APIs, enabling:
  - Easy migration for existing projects
  - Standard API format (REST, JSON)
  - Familiar authentication patterns

## Excluded Providers

Providers are excluded if they meet ANY of the following:

- **Cracked services**: Any service offering unauthorized access to paid AI APIs or models

- **Stolen accounts**: Services based on or promoting unauthorized account access

- **Unauthorized bypass tools**: Tools designed to circumvent:
  - Rate limits
  - Authentication
  - Usage quotas
  - Content filters

- **Misleading "unlimited free" claims**: Providers claiming unlimited access when:
  - Terms explicitly limit usage
  - Service is known to throttle after certain thresholds
  - The claim contradicts official documentation

- **Unsafe or untrustworthy services**: Providers that:
  - Have known security vulnerabilities
  - Engage in deceptive practices
  - Have a history of data misuse
  - Cannot be verified as legitimate businesses

## Quality Standards

In addition to the criteria above, providers should meet these quality standards:

1. **Documentation**: Clear API documentation exists
2. **Reliability**: Service has reasonable uptime
3. **Transparency**: Clear pricing and terms
4. **Accessibility**: Accessible to developers globally (no region lock without disclosure)
5. **Maintenance**: Actively maintained and supported

## Appeals and Corrections

If you believe a provider has been incorrectly categorized or omitted, please:

1. Open an issue on GitHub with evidence
2. Provide documentation supporting the request
3. Allow 14 days for community review

## Criteria Updates

This criteria document is reviewed quarterly. Changes require:

1. Community discussion (minimum 7 days)
2. Pull request approval
3. Maintainer consensus
